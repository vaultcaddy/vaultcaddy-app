# 📋 阶段2：图片间距修复计划

## 🎯 目标

**要求**：所有landing page中，图片与图片之间必须有1000字内容

## 📊 影响范围

### 当前问题
从代码分析发现，许多页面的图片直接相连：

```html
<!-- 图片2 -->
<img src="..." />

<!-- 图片3（直接相连，间距<50字）-->
<img src="..." />
```

### 影响页面统计
1. **银行landing page**：约40个页面
   - ja/hsbc-bank-statement.html
   - ja/hangseng-bank-statement.html
   - ... 等38个

2. **行业解决方案**：约104个页面
   - ja/solutions/restaurant/
   - ja/solutions/retail/
   - ... 等102个

3. **博客文章**：约74个页面
   - ja/blog/xxx.html
   - ... 等74个

**总计**：约218个页面需要修复

## 🔧 解决方案

### 方案设计

**Step 1：内容库设计**

为不同类型的页面创建1000字内容模板：

#### A. 银行页面内容模板
```
模板A1：用户案例（约300字）
模板A2：安全性说明（约200字）
模板A3：API集成教程（约250字）
模板A4：常见问题（约250字）
```

#### B. 行业解决方案页面内容模板
```
模板B1：行业痛点分析（约300字）
模板B2：解决方案详解（约250字）
模板B3：ROI计算（约200字）
模板B4：实施步骤（约250字）
```

#### C. 博客页面内容模板
```
模板C1：深入分析（约300字）
模板C2：实战技巧（约250字）
模板C3：相关工具（约200字）
模板C4：延伸阅读（约250字）
```

### 方案实施

**Step 2：自动修复脚本**

```python
def fix_image_spacing(html_file, page_type):
    """
    自动修复图片间距
    
    Args:
        html_file: HTML文件路径
        page_type: 页面类型（bank/solution/blog）
    
    Process:
        1. 解析HTML，找到所有<img>标签
        2. 检测相邻图片间的文字数量
        3. 如果<1000字，插入对应类型的内容模板
        4. 保持原有HTML结构和样式
    """
```

**Step 3：内容生成策略**

1. **智能内容匹配**
   - 根据页面标题/关键词
   - 选择最相关的内容模板
   - 动态替换占位符

2. **多语言支持**
   - 中文内容：1000字
   - 日文内容：约1500字（考虑日文字符密度）
   - 韩文内容：约1500字
   - 英文内容：约500词

3. **内容质量控制**
   - 确保内容与页面主题相关
   - 包含目标关键词
   - 自然段落分割
   - 包含小标题提升可读性
