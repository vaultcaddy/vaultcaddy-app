# 🔧 打开Console日志 - 3种方法

## 问题原因

生产环境（vaultcaddy.com）自动隐藏了 `console.log` 输出，以保护用户隐私和提升性能。

文件：`disable-console-safe.js` 第24行

---

## ✅ **方法1：浏览器Console中启用（最简单，推荐）**

### 操作步骤（10秒）：

1. **打开Dashboard页面**
   - https://vaultcaddy.com/en/firstproject.html?project=V3UX1IvpVbHLsW2fXZ45

2. **按F12打开DevTools**，切换到 **Console** 标签

3. **在Console中输入以下命令**：
```javascript
enableConsoleLog()
```

4. **按 Enter 键**

5. **看到提示**：
```
✅ 控制台日志已重新启用
```

6. **刷新页面**（Ctrl+R 或 Cmd+R）

7. **重新上传PDF**，现在就能看到详细日志了！

---

## ✅ **方法2：URL添加debug参数（最可靠）**

### 操作步骤：

1. **在当前URL后面添加 `&debug=1`**

原URL：
```
https://vaultcaddy.com/en/firstproject.html?project=V3UX1IvpVbHLsW2fXZ45
```

改为：
```
https://vaultcaddy.com/en/firstproject.html?project=V3UX1IvpVbHLsW2fXZ45&debug=1
```

2. **按 Enter 访问新URL**

3. **打开Console（F12）**，应该看到：
```
🔧 控制台日志已启用（开发环境或debug模式）
```

4. **重新上传PDF**，查看完整日志

---

## ✅ **方法3：临时修改代码（用于深度调试）**

### 修改文件：`disable-console-safe.js`

**当前代码（第24行）**：
```javascript
if (isProduction && !debugMode) {
    // 🔇 生产环境：隐藏console.log
    console.log = function() {
        // 不输出任何内容
    };
```

**改为**：
```javascript
if (false) {  // ✅ 临时禁用日志隐藏
    // 🔇 生产环境：隐藏console.log
    console.log = function() {
        // 不输出任何内容
    };
```

### 上传后：

- 所有Console日志将始终显示
- 用于深度调试后，记得改回 `if (isProduction && !debugMode)`
