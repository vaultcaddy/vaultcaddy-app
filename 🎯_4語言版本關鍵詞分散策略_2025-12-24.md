# 🎯 VaultCaddy 4語言版本關鍵詞分散策略

**創建日期**: 2025年12月24日  
**核心策略**: 通過4個語言版本分散承載100+關鍵詞,避免單頁面過度優化  
**目標**: 每個語言版本排名25-30個核心關鍵詞,總計100+關鍵詞全面覆蓋

---

## 📊 關鍵詞分散策略概覽

### 核心原則

**❌ 錯誤做法**: 在一個頁面堆砌100+關鍵詞
```
結果: 
- 關鍵詞密度過高 → Google判定為關鍵詞堆砌
- 用戶體驗差 → 跳出率高
- 排名低 → 所有關鍵詞都排不上
```

**✅ 正確做法**: 4個語言版本分散承載
```
中文版(zh-HK): 25-30個繁體中文關鍵詞
英文版(en):    25-30個英文關鍵詞  
日文版(jp):    25-30個日文關鍵詞
韓文版(kr):    25-30個韓文關鍵詞

結果:
- 每個頁面關鍵詞密度適中 ✅
- 針對不同市場優化 ✅
- 總計100+關鍵詞覆蓋 ✅
- 排名提升50-100% ✅
```

## 🛠️ 圖片優化技術要求

### 1. 格式選擇

```
優先級:
1. WebP (主格式) - 體積小60%,瀏覽器支援95%
2. AVIF (未來) - 體積更小,但瀏覽器支援80%
3. JPG/PNG (備用) - 確保兼容性

使用picture標籤實現漸進式加載:
```

```html
<picture>
    <!-- 優先載入WebP -->
    <source srcset="/images/hero.webp" type="image/webp">
    <!-- 備用JPG -->
    <img src="/images/hero.jpg" alt="..." loading="lazy">
</picture>
```

### 2. 尺寸響應式

```html
<!-- 不同設備載入不同尺寸 -->
<picture>
    <!-- 桌面版 (>1200px) -->
    <source media="(min-width: 1200px)" srcset="/images/hero-desktop.webp" type="image/webp">
    
    <!-- 平板版 (768-1199px) -->
    <source media="(min-width: 768px)" srcset="/images/hero-tablet.webp" type="image/webp">
    
    <!-- 手機版 (<768px) -->
    <source media="(max-width: 767px)" srcset="/images/hero-mobile.webp" type="image/webp">
    
    <!-- 備用 -->
    <img src="/images/hero-desktop.jpg" alt="..." loading="lazy">
</picture>
```

### 3. 延遲加載

```html
<!-- 首屏圖片: 立即載入 -->
<img src="hero.webp" alt="..." loading="eager">

<!-- 非首屏圖片: 延遲載入 -->
<img src="screenshot.webp" alt="..." loading="lazy">
```

### 5. 圖片壓縮標準

```
Hero Banner (1920x800px):
  - WebP: <150KB
  - JPG: <250KB

產品截圖 (1200x800px):
  - WebP: <100KB
  - PNG: <200KB

流程圖 (1000x600px):
  - SVG: <50KB
  - WebP: <80KB

數據圖表 (800x600px):
  - WebP: <60KB
  - PNG: <120KB
```

**壓縮工具**:
```bash
# 使用ImageMagick壓縮
convert input.jpg -quality 85 -resize 1920x800 output.jpg

# 轉換為WebP
cwebp -q 85 input.jpg -o output.webp

# 批量處理
for file in *.jpg; do cwebp -q 85 "$file" -o "${file%.jpg}.webp"; done
```
