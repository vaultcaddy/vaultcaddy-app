### 5. Schema.org JSON-LD

需要检查的字段：
- `priceCurrency`: HKD → USD/JPY/KRW
- `price`: 对应的价格数字
- `addressCountry`: HK → US/JP/KR
- `inLanguage`: ["zh-HK", "zh-TW", "en"] → 对应语言
- `description`: 完整翻译

## 🔍 快速检查方法

### 方法1：浏览器检查

```bash
# 1. 打开页面
open https://vaultcaddy.com/en/index.html
open https://vaultcaddy.com/jp/index.html
open https://vaultcaddy.com/kr/index.html

# 2. 在每个页面：
# - 查看页面标题（浏览器标签）
# - 搜索页面中的"香港"、"匯豐"、"HK$"等
# - 检查价格是否正确
```

### 方法2：命令行检查

```bash
# 检查英文版是否还有中文
grep -i "香港\|匯豐\|恆生\|HK\$" en/index.html

# 检查日文版是否还有中文
grep -i "香港\|匯豐\|恆生\|HK\$" jp/index.html

# 检查韩文版是否还有中文
grep -i "香港\|匯豐\|恆生\|HK\$" kr/index.html
```

如果有输出，说明还有中文需要清除。

## ⚠️ 常见问题

### Q1: 为什么不能完全自动同步首页？

**A:** 因为首页包含：
- 营销文案（每个市场可能不同）
- SEO优化内容（每个语言的关键词不同）
- 价格策略（可能因市场而异）
- 本地化示例（银行名称、公司名称等）

### Q2: 其他页面可以自动同步吗？

**A:** 可以！其他页面（dashboard, account, billing等）使用：
```bash
./sync-multilingual.sh all
```

### Q3: 我应该多久检查一次？

**A:** 建议：
- **每次修改中文版首页后**：立即检查
- **定期检查**：每月一次
- **发布前**：必须检查
