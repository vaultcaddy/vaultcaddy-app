## 🛠️ 修复方法

### 使用的工具

1. **批量处理脚本**: `remove-learning-center-all.sh`
   - 处理所有主要页面（account, billing, firstproject, dashboard, index）
   - 删除主导航栏和移动端侧边栏中的链接

2. **最终清理脚本**: `remove-learning-center-final.sh`
   - 清理 document-detail, privacy, terms 页面
   - 删除 JSON-LD 结构化数据中的条目

3. **手动精确删除**:
   - 使用 `search_replace` 工具精确删除多行 HTML 块
   - 确保删除干净，不影响其他内容

### 关键删除命令

```bash
# 删除主导航栏（单行）
sed -i '' '/<a href="blog\/" style="color: #4b5563.*學習中心<\/a>/d' file.html

# 删除移动端侧边栏（多行）
sed -i '' '/<a href="blog\/".*fa-graduation-cap/,/<\/a>/d' file.html

# 删除 JSON-LD（多行）
sed -i '' '/{"@type": "SiteNavigationElement",/,/"name": "学習センター"/d' file.html
```

## 🔍 技术细节

### 删除策略

1. **主导航栏**:
   - 直接删除整个 `<a>` 标签
   - 使用 `sed` 单行删除

2. **移动端侧边栏**:
   - 删除包含图标和文本的多行 `<a>` 标签
   - 使用 `sed` 多行匹配删除

3. **JSON-LD**:
   - 删除完整的 SiteNavigationElement 对象
   - 包括逗号，保持 JSON 结构有效

### 为什么不使用 navbar-component.js？

某些页面（如 account.html, billing.html）使用静态 HTML 导航栏，不使用 `navbar-component.js`。因此需要直接修改 HTML 文件。
