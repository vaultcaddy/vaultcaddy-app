# 📊 Xero 整合戰略與匯出決策 (總顧問建議)

**建立日期**: 2026-04-01
**文件作用**: 記錄針對香港市場，我們產品與 Xero 等專業會計軟體的邊界與整合策略。確保開發資源不被過度消耗在 API 串接上，同時最大化產品對會計師的價值。

## 1. 核心問題：我們需要串接 Xero API 嗎？
**總顧問答案：絕對不要！(至少在達到 $20,000 MRR 之前絕對不要)**

### 為什麼不串接 API？
1. **開發成本極高**：Xero API 的認證 (OAuth 2.0)、Token 更新、錯誤處理非常複雜。一人團隊會被困在無止盡的 API 維護中。
2. **用戶授權門檻高**：小老闆通常不知道自己的 Xero 帳號密碼（因為是會計師在管）。如果要求他們在我們的 App 裡登入 Xero 授權，會造成巨大的「轉換摩擦 (Friction)」，流失率會飆升。
3. **責任歸屬問題**：如果我們透過 API 自動把資料寫入 Xero，一旦 AI 分類錯誤或金額辨識錯誤，直接污染了正式帳目，會計師會把責任推給我們，引發嚴重的客訴。

## 2. 最佳解法：出 Xero 可以「一鍵匯入」的 CSV 文件 (The "Magic CSV")

這是最聰明、成本最低、且最受會計師歡迎的做法。

### 運作流程：
1. 老闆在我們的系統點擊「匯出 Xero 格式」。
2. 我們生成一個符合 Xero 官方匯入標準的 `.csv` 檔案。
3. 老闆把這個 `.csv` 檔案 Email 給會計師。
4. 會計師在自己的 Xero 系統裡點擊 "Import"，幾秒鐘內所有單據自動入帳。

### Xero 官方匯入 CSV 必備欄位 (Draft)：
*   `*Date` (日期, 格式如 DD/MM/YYYY)
*   `*Description` (描述/商戶名稱)
*   `*Amount` (金額，支出通常為負數)
*   `Reference` (參考號碼，可放我們生成的單據 ID)
*   `AccountCode` (會計科目代碼，這是進階功能，初期可留空讓會計師在 Xero 內 Map)
*   `TaxType` (稅務類型，香港通常是 "No TAX")

## 3. 戰略優勢 (為什麼會計師會愛死這個 CSV)
*   **掌控權在會計師手上**：會計師可以在匯入 Xero 之前，先用 Excel 打開這個 CSV 快速掃視一眼，確認沒問題再 Import。這給了他們安全感。
*   **零整合成本**：我們不需要跟任何會計師樓的系統做 IT 對接，只要是會計師，都會用 CSV Import 功能。
*   **護城河加深**：當會計師習慣了收到我們這種「一鍵匯入」的 CSV，他們會主動要求其他客戶也去用 VaultCaddy。
